# 超市账单管理系统 —— 毕业设计答辩报告

---

## 一、项目概述

### 1.1 项目背景

随着超市业务规模的不断扩大，传统的手工记账方式已无法满足现代化管理需求。为提高账单管理效率、规范业务流程、增强数据安全性，开发一套专业的超市账单管理系统显得尤为必要。

### 1.2 项目目标

本系统旨在实现超市账单的信息化管理，主要目标包括：
- 实现账单的录入、修改、删除和查询功能
- 实现供应商信息的统一管理
- 实现用户权限分级控制
- 提供数据统计报表功能
- 支持数据导出功能

### 1.3 技术架构

| 层次 | 技术 | 说明 |
|------|------|------|
| 界面层 | Java Swing | 桌面端GUI框架 |
| 数据访问层 | JDBC | Oracle数据库连接 |
| 数据库 | Oracle 12c | 关系型数据库管理系统 |
| 构建工具 | Maven | 项目依赖管理 |

### 4.2 核心代码实现

#### 4.2.1 数据库连接工具类（DBUtil.java）

**功能**：封装Oracle数据库连接和资源释放操作

```java
public class DBUtil {
    private static final String URL = "jdbc:oracle:thin:@127.0.0.1:1521:ORCL";
    private static final String USER = "scott";
    private static final String PASSWORD = "********";

    static {
        Class.forName("oracle.jdbc.driver.OracleDriver");
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PASSWORD);
    }

    public static void close(Connection conn, PreparedStatement pstmt, ResultSet rs) {
        try {
            if (rs != null) rs.close();
            if (pstmt != null) pstmt.close();
            if (conn != null) conn.close();
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
```

**设计说明**：
- 使用静态代码块加载Oracle驱动
- 提供获取连接和关闭资源的静态方法
- 重载close方法支持不同场景

#### 4.2.3 多条件查询实现（BillDao.java）

```java
public List<Bill> search(String productName, Integer isPaid){
    StringBuilder sql = new StringBuilder("SELECT * FROM bill WHERE 1=1");
    List<Object> params = new ArrayList<>();

    if (productName != null && !productName.isEmpty()){
        sql.append(" AND product_name LIKE ?");
        params.add("%" + productName + "%");
    }
    if(isPaid != null){
        sql.append(" AND is_paid = ?");
        params.add(isPaid);
    }
    sql.append(" ORDER BY id DESC");

    ps = conn.prepareStatement(sql.toString());
    for (int i = 0; i < params.size(); i++){
        ps.setObject(i + 1, params.get(i));
    }

    rs = ps.executeQuery();
    // ... 封装结果
}
```

**设计说明**：
- 使用StringBuilder动态拼接SQL语句
- 通过占位符?防止SQL注入攻击
- 支持条件的灵活组合

#### 4.2.4 批量删除与事务管理（BillDao.java）

**功能**：支持多条账单的批量删除，使用事务保证原子性

```java
public boolean delete(int[] ids){
    conn.setAutoCommit(false);

    String sql = "DELETE FROM bill WHERE id = ?";
    ps = conn.prepareStatement(sql);

    for (int id : ids){
        ps.setInt(1, id);
        ps.addBatch();
    }
    ps.executeBatch();
    conn.commit();
    return true;
}
```

**设计说明**：
- 关闭自动提交，开启手动事务
- 使用addBatch()批量添加删除语句
- 使用executeBatch()一次性执行
- 事务提交保证操作的原子性

## 六、项目亮点

### 6.1 数据库规范化设计

系统采用数据库规范化设计原则，账单表通过`supplier_id`外键关联供应商表，而非直接存储供应商名称。这种设计的优势在于：
- 避免数据冗余，同一供应商信息只存储一份
- 修改供应商信息时自动同步到所有关联账单
- 保证数据一致性，防止因手动输入导致的名称不一致

### 6.2 供应商下拉选择机制

在`BillDialog`中巧妙实现了供应商下拉选择功能：下拉框存储完整的Supplier对象，但通过自定义渲染器只显示供应商名称。保存时提取供应商ID存入数据库，既方便用户操作，又符合数据库设计规范。

### 6.3 灵活的多条件查询

账单和用户查询模块支持多条件组合查询，使用StringBuilder动态拼接SQL语句，通过占位符参数化查询防止SQL注入攻击，实现了灵活、安全的查询功能。

### 6.4 事务与批量操作

批量删除功能使用数据库事务保证操作的原子性，通过addBatch()和executeBatch()实现批量执行，提高了操作效率和数据安全性。

### 6.5 基于角色的权限控制

系统实现了基于角色的菜单权限控制，普通员工只能访问账单管理功能，部门经理拥有全部权限，满足了企业级应用的权限管理需求。

### 6.6 数据导出与报表统计

系统支持将账单数据导出为CSV文件，并提供直观的报表统计功能，包括总交易金额、已付款/未付款金额统计以及供应商交易排行，为管理者提供决策支持。

## 七、不足之处

### 7.1 数据库连接未使用连接池

当前实现每次数据库操作都创建新连接，频繁的连接创建和销毁会造成性能损耗，且无法控制连接数量。

### 7.2 密码明文存储

用户密码以明文形式存储在数据库中，存在安全隐患，一旦数据库泄露将导致用户密码暴露。

### 7.3 SQL查询效率问题

在账单列表加载时，采用N+1查询模式（先查询所有账单，再逐个查询供应商名称），数据量大时会影响性能。此外，修改模式下查询所有数据再遍历查找，效率较低。

### 7.4 异常处理不完善

多处异常仅打印堆栈信息，未进行友好的用户提示或日志记录，用户无法了解操作失败的具体原因。

### 7.5 缺乏分页功能

账单和用户列表未实现分页功能，数据量大时会影响加载速度和用户体验。

### 7.6 未使用Service层

业务逻辑直接写在UI层，违反了分层架构原则，不利于代码维护和单元测试。

## 八、后续优化方向

### 8.1 引入数据库连接池

建议引入Druid或HikariCP连接池，配置最小/最大连接数，复用数据库连接，提高系统性能。

### 8.2 密码加密存储

使用BCrypt或SHA256等加密算法对用户密码进行加密存储，登录时验证加密后的密码，确保用户信息安全。

### 8.3 SQL优化

- 使用JOIN查询替代N+1查询模式
- 添加findById方法直接查询单条记录
- 为频繁查询的字段添加索引

### 8.4 完善异常处理

引入日志框架（如Logback）记录异常信息，同时给用户友好的错误提示，提升用户体验。

### 8.5 实现分页功能

基于Oracle的ROWNUM实现分页查询，控制每页显示的数据量，提高列表加载速度。

### 8.6 添加Service层

将业务逻辑从UI层抽离到Service层，实现业务逻辑与界面展示的解耦，便于单元测试和代码维护。

### 8.7 界面美化

利用项目中已有的图片资源美化界面，添加按钮图标、背景图片等，提升系统视觉效果。
